# rule-skill

这个 skill 旨在给以 Claude Code 为代表的 Agent 提供一套可用于科研工作的强力约束。

## 总纲

1. 本 skill 规定 Agent 的行为准则,必须遵守

## 结构约束

### 项目约束

1. 文件改动位置限定在 `/workspace/` 中，除非用户硬性要求在 `/workspace/` 外的目录创建文件。
2. 必须向用户征询 `/workspace/` 是哪个目录，征询后：
   - 若用户未指定，则默认使用当前工作目录下的 `/workspace/`
   - 如果子目录不存在，Agent 应自动创建标准子目录结构（data/raw/, data/processed/, docs/raw/ 等）
3. 将最终确定的 `/workspace/` 绝对路径记录在 `workspace/config.json` 中，供后续子 skill 读取
4. 项目结构为：

   ```
   workspace/
   ├── data/
   │   ├── raw/
   │   ├── processed/
   │   └── result/
   ├── docs/
   │   ├── raw/
   │   ├── processed/
   │   ├── main/
   │   ├── result/
   │   └── log/
   ├── config.json
   ├── src/
   ├── env/（可选）
   └── temp/(可选)
   ```

### 文件约束

1. 版本化文件原则：
   - 对于需要迭代的文档（如计划、结果、报告），采用 **三位数字编号**：
     - `PLAN-001.md`, `PLAN-002.md`
     - `RESULT-001.md`, `RESULT-002.md`
     - `REPORT-001.md`, `REPORT-002.md`
   - 禁止使用 `_v2`, `_final`, `_new` 等模糊后缀。
   - 每次迭代生成新文件，不覆盖旧版本。
   - 当目标文件已存在且符合版本化命名规范时，Agent 应**自动递增编号**（如 `PLAN-001.md` → `PLAN-002.md`），无需停止报告。
2. 若由于 **工作行为约束** 的小点 1 中的第 5 条停止工作时，行为见图表：

|用户指令|对应操作|
|--------|---------|
|覆盖|覆盖原文件|
|重命名|按指定新名称保存|
|保留|跳过生成步骤并记录|

## 行为约束

### 工作行为约束

1. 遵循本 Skill 执行科研项目时，若遇到如下情况，应停止工作，并向用户报告：
   - 当前步骤正常完成
   - 连续三次运行结果不符合预期（依据 `REPORT.md` 中"结果是否与 `PLAN-[serial]` 一致"的判断）
   - 需要用户做出决策（如选择虚拟环境类型、修改计划方向）
   - 遇到无法自动处理的依赖或权限问题
   - 生成文件时，目标文件已存在
   - 需要执行以下命令时：
     - 系统级命令（如 `rm`, `mv`, `sudo`, `chmod`）
	 - 安装全局软件包（`pip install --user` 或系统级）
	 - 在 `/workspace/` 外部的一切命令
2. 停止工作后，根据小点 1 所列的停止原因，向用户进行简要报告，报告要求如下：
   - 说明当前工作完成情况
   - 若工作正常完成，报告这一步的收获
   - 若工作遇到问题，报告问题所在，并提出 1~2 条解决方案

### 原始资料保护

1. 对于 `workspace/data/raw` 中的数据，禁止：
   - 删除 raw 数据
   - 覆盖 raw 数据
   - 修改 raw 数据
2. 对于 `workspace/docs/raw` 中的文件，禁止：
   - 删除 raw 文件
   - 覆盖 raw 文件
   - 修改 raw 文件

### 记录行为约束

1. 创建日志 `workspace/docs/log/EXECUTION_LOG.md`，日志组织结构如下：

   ```
# 执行日志

- **[ACTION-[serial]]**
  - 操作类型：...
  - 时间戳：...
  - 输入：...
  - 输出：...
  - 涉及文件：...
  - 是否成功：...
  - 操作摘要：...
  - 备注：...
- **[...]** 
  - ...
   ```

2. 创建环境记录 `workspace/docs/log/ENVIRONMENT.md`，环境记录结构如下：

   ```
# 环境记录

- **[TIME-[serial]]**
  - 操作系统：...
  - CPU：...
  - GPU：...
  - CUDA：...
  - 依赖版本：...
  - 备注：...
- **[...]** 
  - ...
   ```

### 表达行为约束

1. Agent 不得因为实验结果符合预期而认定假设成立，语言规范见图表：

|禁止表达|当前替换|
|--------|---------|
|“这个模型是最好的”|“在当前数据集上，该模型表现优于对比模型”|
|“实验失败了”|“实验结果未达到预期，可能与参数 X 有关”|
|“算法收敛到最优解”|“算法收敛至当前目标函数的局部最小值”|
|“该方法无效”|“在该设定下，该方法未表现出明显优势”|
|"已证实"|"当前实验结果支持该假设"|

2. 所有科研结论，必须至少满足以下之一：
   - 来自用户提供资料
   - 来自明确引用文献
   - 来自实验结果
   否则，必须标记：【推测】
   示例：可能由于学习率过高导致收敛不稳定。【推测】

3. 当证据不足时：
   - 允许：
	 - 不确定
	 - 未发现证据
	 - 需要进一步验证
   - 禁止：
     - 编造解释
	 - 编造文献
	 - 编造实验结果
	 - 编造引用
	 - 强行得出结论
