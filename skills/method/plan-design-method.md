# plan-design-method

这个 skill 旨在给以 Claude Code 为代表的 Agent 提供一套能够用于科研计划制定的方法论。

## 相关 skill 加载

1. 在进行 **计划生成前提** 之前，必须加载 `plan-design-constraint.md` 

## 计划生成前提

1. 在生成计划前，必须确认：
   - 用户科研目标已明确；
   - 已读取并理解 `SUMMARY.md`（若存在）；
   - 已读取并理解 `RESULT`（若存在）；
   - 已读取并理解 `REPORT`（若存在）；
   - 若上述信息不足以支撑计划制定，应先向用户提问，不得直接生成计划。
2. Agent 不得擅自扩展用户未提出的科研目标。
   - 若认为需要扩展研究范围，应说明理由并征求用户同意。
   - 未获得批准前，不得写入计划。

---

## 计划文件管理

1. 在 `workspace/docs/main/` 中创建计划文件：
2. 当前活跃计划（Active Plan）应始终为最新批准版本。

---

## 用户审批机制

1. Agent 完成计划后：
   - 向用户展示计划摘要；
   - 说明主要目标；
   - 说明风险点；
   - 说明 Baseline；
2. 等待用户批准。
3. 用户批准后,允许进入： `PLAN_IMPLEMENTING` 状态。
