# data-process-method

这个 skill 旨在给以 Claude Code 为代表的 Agent 提供一套关于科研数据清洗与预处理的方法论。

本 skill 遵循 `rules.md` 及 `regulation/` 下的所有约束规范。原始数据保护参见 `data-protect-constraint.md`（若已拆分），日志记录见 `log-write-method.md`。

---

## 总原则

1. **永远保留原始数据**：所有清洗操作必须生成新文件，不得覆盖 `data/raw/` 中的原始数据。
2. **所有操作可追溯**：每一步清洗操作都应记录在日志中，确保可复现。
3. **先探索，后清洗**：在动手清洗之前，必须先理解数据的全貌。

---

## 数据清洗流程

### 第一步：初步探索

在开始清洗之前，先快速了解数据的基本情况：

1. 使用 `df.info()` 查看字段类型、记录总数、各列缺失情况。
2. 使用 `df.describe()` 查看数值型列的分布统计（均值、标准差、分位数等）。
3. 绘制直方图或箱线图，直观发现异常值分布。
4. 记录探索结果到日志，为后续清洗提供依据。

### 第二步：处理缺失值

根据缺失情况选择合适策略：

**删除策略：**
- 若某列缺失比例 > 70%，且该列对分析不重要，删除整列。
- 若缺失记录随机分布且样本量充足，可直接删除含缺失的行。

**填充策略：**
- 连续变量：使用均值填充（数据近似正态分布）或中位数填充（数据偏态分布）。
- 分类变量：使用众数填充。
- 时间序列数据：使用插值法（线性插值、时间插值等）。
- 若数据量足够，可构建简单模型预测缺失值。

### 第三步：处理异常值

**异常值检测方法：**
- **统计方法**：超过均值 ± 3 倍标准差的值，或箱线图中低于 Q1 - 1.5×IQR 或高于 Q3 + 1.5×IQR 的值。
- **领域知识**：结合具体研究领域判断，如年龄 > 120、血压为负值等明显不符合常识的值。

**处理方式：**
1. 先回溯实验记录或数据来源，确认该异常值是否为数据错误（如录入错误、传感器故障）。
2. 若是数据错误：修正（如可确定正确值）或删除（如无法确定正确值）。
3. 若是真实的极端值（如某疾病患者的特殊指标），**应保留**，但在报告中注明其存在及处理依据。

### 第四步：标准化数据格式

**格式统一：**
- 日期字段：统一转为 `YYYY-MM-DD` 格式。
- 分类变量：统一大小写（如 `Male`/`male` → `male`）。
- ID 类字段：转为字符串类型，避免被误当作数值参与计算。

**类型纠正：**
- 检查数值型列是否被误存为文本（如 `'5.2'` → `5.2`）。
- 检查逻辑范围：如年龄不应超过 150、收缩压不应低于舒张压等。

### 第五步：去重与逻辑校验

**去重：**
- 完全重复的行：直接删除。
- 部分重复（如基于患者 ID + 检测时间判断）：保留最新记录。

**逻辑校验：**
- 检查字段间的逻辑一致性：如体重下降但记录显示“食欲大增”，需标记。
- 检查范围合理性：如收缩压不应低于舒张压。

### 第六步：文档记录

1. 每一步清洗操作都应在代码中记录（推荐使用 Jupyter Notebook 或 Python 脚本）。
2. 清洗完成后，将清洗后的数据另存为新文件（如 `data_processed.csv`），不覆盖原始数据。
3. 在日志中记录：
   - 清洗前后的数据行数变化。
   - 具体处理了哪些字段、使用了什么方法。
   - 处理后的数据保存位置。
