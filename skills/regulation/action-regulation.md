# action-regulation

这个 skill 旨在给以 Claude Code 为代表的 Agent 提供一套可用于科研活动中实际工作的规范。

## 工作行为约束

### 停止机制

1. 遵循本 Skill 执行科研项目时，若遇到如下情况，应停止工作，并向用户报告：
   - 当前步骤正常完成
   - 连续三次运行结果不符合预期（依据 `REPORT-[serial]` 中 "结果是否与 `PLAN-[serial]` 一致" 的判断）
   - 需要用户做出决策（如选择虚拟环境类型、修改计划方向）
   - 遇到无法自动处理的依赖或权限问题
   - 生成文件时，目标文件已存在
   - 需要执行以下命令时：
     - 系统级命令（如 `rm`, `mv`, `sudo`, `chmod`）
     - 安装全局软件包（`pip install --user` 或系统级）
     - 在 `/workspace/` 外部的一切命令
	 
### 报告机制

1. 停止工作后，根据第 1 条所列的停止原因，向用户进行简要报告，报告要求如下：
   - 说明当前工作完成情况
   - 若工作正常完成，报告这一步的收获
   - 若工作遇到问题，报告问题所在，并提出 1~2 条解决方案

### 记录机制

1. 参考 `log-write-constraint`，创建日志 `EXECUTION_LOG.md`
2. 参考 `log-write-constraint`，创建日志 `ENVIRONMENT_LOG.md`
