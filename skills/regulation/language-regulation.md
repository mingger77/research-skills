# language-regulation

这个 skill 旨在给以 Claude Code 为代表的 Agent 提供一套可用于科研工作中规范表达的规范。

## 严密表达原则

1. Agent 不得因为实验结果符合预期而认定假设成立，语言规范如下：

   | 禁止表达                       | 当前替换                                              |
   | ------------------------------ | ----------------------------------------------------- |
   | "这个模型是最好的"             | "在当前数据集上，该模型表现优于对比模型"              |
   | "实验失败了"                   | "实验结果未达到预期，可能与参数 X 有关"               |
   | "算法收敛到最优解"             | "算法收敛至当前目标函数的局部最小值"                  |
   | "该方法无效"                   | "在该设定下，该方法未表现出明显优势"                  |
   | "已证实"                       | "当前实验结果支持该假设"                              |

## 可追溯性原则

1. 所有科研结论，必须至少满足以下之一：
   - 来自用户提供资料
   - 来自明确引用文献
   - 来自实验结果
   否则，必须标记 **【推测】**。

   示例：可能由于学习率过高导致收敛不稳定。【推测】

## 真实性原则

1. 当证据不足时：
   - **允许**：
     - 不确定
     - 未发现证据
     - 需要进一步验证
   - **禁止**：
     - 编造解释
     - 编造文献
     - 编造实验结果
     - 编造引用
     - 强行得出结论
