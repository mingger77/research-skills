# report-write-constraint

这个 skill 旨在给以 Claude Code 为代表的 Agent 提供一套能够用于在科研中进行报告撰写的方法约束。

## 报告结果定性

1. `REPORT` 必须判断结果是否与`PLAN` 中的目标一致
   - 若一致，用 1~2 句话总结结论
   - 若不一致，参见 **失败处理**
	 
## 报告结果解释

1. `REPORT` 中每项主要结论必须标注：
   - High：
   多次实验 + 多来源支持
   - Medium：
   单次实验或部分支持
   - Low：
   推测性结论、
2. `REPORT` 中的每个重要结论，必须能够追溯到以下任意一个文件的具体实验记录：
   - RESULT
   - EXPERIMENT_LOG
3. `REPORT` 中不得出现无法定位来源的描述。
4. 若结果包含图表, `REPORT` 必须说明：
   - 图表反映的趋势
   - 是否符合预期
   - 是否存在异常点
   - 对科研目标的意义

## 报告内容补充

1. 若 `PLAN` 中存在 Baseline 方案， `REPORT` 中必须增加：
   - Baseline结果
   - 当前方案结果
   - 差异分析
   不得仅汇报当前方案结果。
2. `REPORT` 末尾增加 Future Work 部分，内容包括：
   - 推荐改进方向
   - 可能新增数据
   - 可能新增模型
   - 潜在风险

## 失败处理

1. 实验未达到预期时，必须在 `REPORT` 中记录：
   - 使用方案
   - 参数
   - 失败现象
   - 可能原因
   不得直接丢弃。
2. `FINAL_REPORT.md` 应包含
   - 已尝试方案
   - 失败原因
   - 当前瓶颈

## 未知结果归因

2. `REPOET` 必须从以下四点说明不确定性与局限性：
   - 当前实验规模限制
   - 数据集限制
   - 参数设置限制
   - 尚未验证的问题
