# result-write-constraint

这个 skill 旨在给以 Claude Code 为代表的 Agent 提供一套能够用于在科研中进行结果编写的方法约束。

1. `RESULT` 至少包含：
   - 每次运行前自动记录所用参数（包括随机种子）
   - 是否成功运行（exit code）
     - 若成功运行，记录：
	   - 核心结果（数值、图片、表格）
	   - 核心结果所在位置
	 - 若实验失败，记录：
	   - 错误类型
       - 错误信息摘要
       - 是否可复现
       - 可能原因
2. 如果同一轮实验中包含多组运行（如参数扫描、多随机种子），应在同一个 `RESULT` 中以表格或列表形式汇总所有运行记录，而非生成多个结果文件
3. 若 `PLAN` 中存在 Baseline 方案， `RESULT` 中必须增加：
   - Baseline结果
   - 当前方案结果
   - 差异分析
   不得仅汇报当前方案结果。
