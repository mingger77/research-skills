# Research-with-Python-Skill

这个 Skill 旨在给以 Claude Code 为代表的 Agent 提供一套可用于科研的 Python 程序编写方法论。

**总原则：安全 > 效率**

**使用场景说明**：本 Skill 适用于 **探索性科研编程任务**，不适用于生产环境部署或实时系统。

---

## 总纲

1. 简述：
   - 总纲规定 Agent 的行为准则
   - 准备篇规定 **方向确定**、**资料收集及整合** 和 **计划撰写** 的方法
   - 实行篇规定 **Python 程序编写**、**结果分析与验证** 的方法
2. 在正式进行 **准备篇** 和 **实行篇** 的每一步之前，必须停下来等待用户指示。
3. 文件改动位置限定在 `/workspace/` 中，除非用户硬性要求在 `/workspace/` 外的目录创建文件。
4. 项目结构为：

   ```
   workspace/
   ├── data/
   │   ├── raw/
   │   └── processed/
   ├── docs/
   │   ├── SUMMARY.md
   │   ├── PLAN.md
   │   ├── REPORT.md
   │   └── EXPERIMENT_LOG.md
   ├── src/
   ├── result/
   └── env/
   ```

5. 遵循本 Skill 执行科研项目时，若遇到如下情况，应停止工作：
   - 当前步骤正常完成
   - 连续两次运行结果不符合预期（依据 `REPORT.md` 中"结果是否与 `PLAN.md` 一致"的判断）
   - 需要用户做出决策（如选择虚拟环境类型、修改计划方向）
   - 遇到无法自动处理的依赖或权限问题
6. 停止工作后，根据第 5 条所列的停止原因，向用户进行简要报告，报告要求如下：
   - 说明当前工作完成情况
   - 若工作正常完成，报告这一步的收获
   - 若工作遇到问题，报告问题所在，并提出 1~2 条解决方案
7. Agent 不得因为实验结果符合预期而认定假设成立，语言规范如下：
   - **禁止措辞**（不得使用以下示例相同或类似的强结论措辞）：
     - 证明
     - 必然说明
     - 已证实
   - **推荐措辞**（使用与以下示例相同或类似的保守措辞）：
     - 初步
     - 当前实验结果支持该假设
     - 当前实验支持该观点

## 准备篇

### 方向确定

1. 在正式开始工作之前，向用户询问以下信息：
   - 用户的研究方向
   - 用户的科研目标
   - 用户是否有相关领域的资料

### 资料收集及整合

1. 根据用户是否提供资料及资料格式，作出如下回应：
   - 如果用户没有资料，直接跳到第 2 步
   - 如果用户提供的资料是 Markdown 格式，直接采纳
   - 如果用户提供的资料不是 Markdown 格式，应将其转换为 Markdown 格式
2. 结合用户研究方向及用户提供的资料，在网上搜寻相关资料。资料来源分级如下：
   - **A 级**：
     - Nature、Science、IEEE、ACM、Springer
   - **B 级**：
     - Arxiv、Google Scholar、中国知网
   - **C 级**：
     - 技术博客、CSDN
3. 从网上搜索的资料必须附上来源网址。
4. 若没有联网搜索的能力，应向用户说明相关领域高质量资料的来源及获取方式。
5. 将搜寻的资料（或用户根据第 4 步引导查找到的资料）写在若干 Markdown 文档中。
6. 原始数据放在 `workspace/data/raw/` 中；处理后数据放在 `workspace/data/processed/` 中。
7. 将所有资料的重点内容总结到 `SUMMARY.md` 中。

### 计划撰写

1. 根据用户的科研目标及 `SUMMARY.md` 制定实施计划，命名为 `PLAN.md`。
2. `PLAN.md` 应包含以下方面的内容：
   - **目标拆解**（子任务、验收标准）：
     - 子任务输入
     - 子任务输出
     - 子任务成功标准
     - 子任务失败判定条件
   - Python 依赖库列表（含版本建议）
   - 核心代码逻辑（伪代码或结构化描述）
   - 可能遇到的困难及备选方案（至少 1 项）
   - Baseline 方案
3. 等待用户批准 `PLAN.md` 后，才能进入 **实行篇**。

---

## 实行篇

### Python 程序编写

1. **虚拟环境准备**：
   - 默认在 `/workspace/env` 创建 venv；若用户要求 conda 则询问环境名
   - 使用 **清华源** 安装依赖（`-i https://pypi.tuna.tsinghua.edu.cn/simple`）
   - 若依赖安装失败，停止并报告
2. **程序编写及运行**：
   - 创建相应的 Python 文件
   - 运行程序，将结果放在 `/result/` 中
   - 针对 Python 文件及其结果编写解析文档
3. **Python 代码要求**：
   - 必须拆分函数
   - 必须包含必要注释
   - 必须提供 `requirements.txt`
   - 必须提供运行说明
   - 必须固定随机数种子

### 结果分析与验证

1. 程序运行后，自动生成 `EXPERIMENT_LOG.md` 和 `RESULT.md`。
2. `EXPERIMENT_LOG.md` 至少包含：
   - 时间
   - Git Commit
   - 参数
   - 随机种子
   - 数据集版本
   - Python 版本
   - 依赖版本
3. `RESULT.md` 至少包含：
   - 每次运行前自动记录所用参数（包括随机种子）
   - 是否成功运行（exit code）
   - 核心结果（数值、图片、表格）所在位置
4. 当用户明确要求"总结"或"结束本轮研究"时，生成 `REPORT.md`。
5. `REPORT.md` 至少包含：
   - 结果是否与 `PLAN.md` 中的目标一致
   - 若一致：用 1~2 句话总结结论
   - 若不一致：
     - 列出至少 2 条可能原因
     - 提出至少 1 条修改建议
   - 当前结果是否能回答 `PLAN.md` 中的科研目标
     - 若能，结束进程
     - 若不能，跳到本小节第 6 步
6. 当 `REPORT.md` 指出"无法回答科研目标"时，Agent 应：
   - 分析现有结果与原始目标的差距
   - 提出 1~2 种改进路径（如：修改参数、更换模型、重新收集数据）
   - 生成 `PLAN_v2.md`
   - 停止并向用户报告："已生成新计划，是否继续？"
   - 用户批准后，重复执行 **实行篇**（从虚拟环境或代码修改开始，可复用之前环境）
7. 除非用户要求，否则 **不自动修改代码**，只报告问题。
